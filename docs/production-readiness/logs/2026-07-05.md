1. Stack independence is still an unproven readiness claim.

   Evidence: `docs/improvement-playbook.ja.md` B-4 requires one dissimilar
   repository run, and `docs/improvement-report-2026-07-03.ja.md` P4-(b)

   record a completed non-Node Issue-to-PR-to-merge run.

   evidence that builder, verifier, workspace setup, or verification-command
   contracts work on external or non-Node products.

   fixture repository, add `.kaizen/config.yml`, complete one
   Issue-to-PR-to-merge run, then file every discovered stack-assumption failure
   in the target repository.
